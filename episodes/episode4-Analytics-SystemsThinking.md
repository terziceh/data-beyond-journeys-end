### 2. ArcGIS Light Infrastructure Dashboard (Completed)

* Built an interactive geospatial dashboard comparing 2016 vs 2025 light infrastructure data
* Cleaned and integrated multiple datasets with inconsistent formats

**Key Data Work:**

* Identified mismatch between total rows (389) and unique poles (112)
* Filtered for VCU-owned infrastructure only
* Standardized schema across datasets
* Rebuilt consistent Pole_ID system
* Prepared latitude/longitude for mapping

**Dashboard Features:**

* Interactive map of light pole locations
* Time-based comparison between datasets
* Campus-level filtering (MCV vs MPC)
* Infrastructure change visualization

## 💡 Major Realizations

### 1. Data > Visualization

The hardest part wasn’t building the dashboard—it was getting the data into a usable state. Visualization became straightforward once the structure was correct.

---

### 2. Real Data Is Messy

The 2016 dataset highlighted how unreliable real-world data can be. Duplicate identifiers and inconsistent records required rebuilding logic rather than trusting the source.

---

### 3. Small Details Matter

Things like:

* consistent IDs
* correct grouping logic
* filtering rules

made a big difference in whether the output was actually meaningful.

---

### 4. Thinking Like a Builder > Analyst

Shift started from:

* “what chart should I make?”

to:

* “how should this system work end-to-end?”
